# Merchant Evidence Scorecard

This scorecard evaluates the validity of the problems claimed by the merchant.

## Evidence Scale
- **0** — Opinion only (e.g., "I think we lose sales").
- **1** — Vague recalled problem (e.g., "Yeah, last week someone got mad").
- **2** — Specific recent incident (e.g., "On Tuesday, a guy asked for an Ergo One, we replied 3 hours later, and he bought elsewhere").
- **3** — Workflow or artifact shown (e.g., Merchant shares screen and shows the exact lost conversation).
- **4** — Data/shared samples or pilot commitment (e.g., Merchant exports chat history for analysis).
- **5** — Payment or paid-pilot commitment (e.g., Merchant pays deposit to solve the problem).

## Scoring Matrix
For each interview, score the following claims using the 0-5 scale:

1. **Pain: Slow First Response**
   - Evidence Score: [0-5]
   - Financial Consequence: ___
2. **Pain: Repetitive Catalog Questions**
   - Evidence Score: [0-5]
   - Time Cost: ___
3. **Pain: Lost / Forgotten Follow-ups**
   - Evidence Score: [0-5]
   - Financial Consequence: ___
4. **Pain: Purchase Handoff Friction**
   - Evidence Score: [0-5]
   - Operational Consequence: ___
5. **Commitment to Test**
   - Evidence Score: [0-5]
6. **Willingness to Pay**
   - Evidence Score: [0-5]

*Note: Any merchant scoring below a 3 on core pain points should be excluded from the Concierge Pilot.*
